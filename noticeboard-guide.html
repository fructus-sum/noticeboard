01

Overview

Noticeboard turns one or more Raspberry Pis into a managed digital display system. One Pi runs the server and admin panel. Any number of additional Pis (or browsers on a local network) can act as display screens — showing slideshows of images, videos, and text slides that you control remotely from the admin panel.

How it works

The system has three parts that work together:

PartWhat it doesWho uses it
Server Runs on one Pi. Stores all content, manages slideshows, and serves both the admin panel and display pages. Always running in the background
Admin Panel A web page (at /admin) where you log in, create slideshows, upload images, add text slides, and control what appears on screen. You — the administrator
Display A browser page (at /) that shows the active slideshow. Can run on the server Pi itself or on separate display Pis. The audience — no interaction needed

When you make a change in the admin panel, the display screens update automatically within seconds — no page refresh required.

ℹ
The display screen requires no interaction and is designed to run unattended. Once set up, the Pi running the display can be left alone indefinitely.
02

Installation

There are two installer scripts depending on your setup. Both require a Raspberry Pi running Raspberry Pi OS (Bookworm or Bullseye recommended) with a desktop environment.

Option A — Single Pi (server + display on the same Pi)

Use this if you have one Pi that will both run the server and show the display on its own screen.

Open a terminal on the Pi

Press Ctrl+Alt+T, or find Terminal in the application menu.

Download and run the installer

Copy and paste this command, then press Enter. You will be asked for your password.

Wait for completion

The installer will download Node.js, clone the project, build the web app, and set up a background service. This takes 3–8 minutes depending on your internet connection.

Reboot the Pi

Run sudo reboot. After restarting, Chromium will open automatically in fullscreen kiosk mode showing the display.

curl -fsSL https://raw.githubusercontent.com/fructus-sum/noticeboard/main/installers/installer-server-local.sh | sudo bash
⚠
Change your password after first login. The default admin password is Admin@12345. Anyone on your local network can reach the admin panel, so changing this immediately is important. See Section 3.

Option B — Separate display Pi (remote client)

Use this if you already have a server Pi running and want to add an additional Pi as a second (or third) display screen.

Find the server Pi's IP address

On the server Pi, open a terminal and run hostname -I. Note the first address shown (e.g. 192.168.1.42).

Run the remote installer on the display Pi

On the new display Pi, open a terminal and run the command below. When prompted, enter the server URL.

Enter the server URL when prompted

You will be asked: "Noticeboard server URL" — enter it in the format http://192.168.1.42:3000 (use the IP from step 1).

Approve the display in the admin panel

If MAC filtering is enabled on the server, the new display will show its MAC address on screen. You must add it to the approved list in Admin → Settings before it can show content. See Section 11.

Reboot the display Pi

Run sudo reboot. Chromium will open automatically and connect to the server.

curl -fsSL https://raw.githubusercontent.com/fructus-sum/noticeboard/main/installers/installer-remote-client.sh | sudo bash

Accessing the admin panel after installation

Open any browser on your network and go to:

http://<your-pi-ip>:3000/admin
Replace <your-pi-ip> with the Pi's local IP address — e.g. http://192.168.1.42:3000/admin

On the Pi itself, you can always use http://localhost:3000/admin
03

First Login

Logging in

Open the admin panel

Navigate to http://<pi-ip>:3000/admin in your browser.

Enter the default credentials

Password: Admin@12345 — this is the same for all fresh installations.

Change the password immediately

Go to Settings (top-right of the admin panel) and set a strong password before doing anything else.

🔒
Security notice: The admin panel is accessible to any device on your local network. Always change the default password. The display page (/) does not require a password and is intentionally public on the local network so that any screen can show it.

Changing the admin password

Click "Settings" in the admin navigation bar
Scroll to the "Change Password" section

Enter your current password, then your new password twice. Click Save.

You will be logged out automatically

Log back in with your new password to confirm it worked.

Session behaviour

Admin sessions are stored as secure cookies and last until you log out or the session expires. You do not need to log in again after a browser refresh. If you get a "Session expired" message, simply log in again.

04

Creating Slideshows

A slideshow is a named collection of slides — images, videos, and text slides — that plays on the display screen. You can have multiple slideshows and control which ones are active, when they appear, and in what order.

Creating your first slideshow

Go to Slideshows in the admin panel

Click "Slideshows" in the navigation bar at the top of the page.

Click "+ New slideshow"

A form will appear asking for a name.

Enter a descriptive name

Choose something meaningful like "Main Notices", "Weekly Menu", or "Events This Month". The name is only visible to you in the admin panel.

Click Create

The slideshow is created and appears in the list. It starts in Disabled status — it won't appear on the display until you publish it.

ℹ
New slideshows always start as Disabled. This lets you upload all your content and get everything set up before it appears on screen. When you're ready, click Publish. See Section 8.

Managing an existing slideshow

Click Manage next to any slideshow in the list to open its detail page. From here you can:

ActionHow
Rename the slideshowClick Edit in the Settings card, change the name, click Save
Change the scheduleClick Edit, change Schedule type, click Save
Change the priorityClick Edit, adjust the Priority number, click Save
Publish or disableClick the Publish / Disable button in the Status field
Upload image or video slidesClick + Upload in the Slides card — select one or more files
Create a text slideClick T+ Text Slide — see Section 6
Add text overlay to a slideClick the ✏ button on any ready slide — see Section 6
Upload a watermark / logoUse the Watermark / Logo card — see Section 7
Reorder slidesUse the ↑ ↓ arrow buttons next to each slide
Delete a slideClick the ✕ button next to the slide

Deleting a slideshow

From the slideshow list, click Delete next to the slideshow. You will be asked to confirm. Deletion is permanent — all uploaded media files for that slideshow will also be removed from the server.

05

Uploading Media

Each slideshow holds an ordered list of slides. A slide can be an image, a video clip, or a text slide (see Section 6). You upload image and video files from the slideshow's detail page — you can select multiple files at once.

Supported file types

TypeAccepted formatsMax size
Images JPEG, PNG, GIF, WebP 500 MB per file
Videos MP4, MOV, AVI, WebM, MPEG 500 MB per file
ℹ
Images are automatically converted to PNG for consistent display. Videos are re-encoded to H.264 MP4 for reliable playback in Chromium. On a Raspberry Pi 4 this process takes 10–60 seconds per file depending on size. Each slide will show processing during this time.

Uploading slides

Open the slideshow's detail page

Click Manage next to the slideshow in the list.

Click "+ Upload" in the Slides section

A file picker will open. You can select multiple images or videos at once — hold Ctrl (or Cmd on Mac) to select more than one file.

Wait for the slides to process

Each uploaded file appears in the list with a processing badge. This changes to ready when conversion is complete. The page updates automatically — no need to refresh.

Upload more at any time

You can click Upload again immediately without waiting for the previous batch to finish.

Slide status meanings

How long each slide displays

Images and text slides display for a fixed number of seconds. The default is 10 seconds. You can change the global default in Settings → Default slide duration. Text slides also have their own per-slide duration you can set when creating or editing them.

Videos play from start to finish at their natural length before moving to the next slide.

Reordering slides

On the slideshow detail page, each slide has ↑ and ↓ buttons. Click these to move a slide up or down in the running order. The display will update to the new order within a few seconds.

Text Overlays & Text Slides

You can add text directly on top of any slide or create a completely blank text-only slide — like a PowerPoint slide — entirely from the admin panel. No image editing software needed.

Text overlays — adding text on top of an image or video

A text overlay places a styled message on top of an existing image or video slide. It appears on the display at the position you choose, and updates in real time as you adjust the controls.

Open the slideshow detail page

Click Manage next to the slideshow.

Click the ✏ button on a ready slide

The slide editor opens. Slides that are still processing cannot be edited yet.

Type your message in the Text field

The live preview above the controls updates instantly as you type.

Adjust the style and position

Use the controls below to set font, size, colour, and position. See the table below for what each control does.

Click Save

The overlay is saved and the display updates automatically. A small purple T badge appears on the slide row to show it has an overlay.

Open the slide editor (✏ button) and tick "Remove overlay from this slide", then click Save. The overlay is deleted and the purple T badge disappears from the slide row.

Text slides — blank slides with coloured backgrounds

A text slide is a completely blank slide with a solid colour background and your text on top — no image file required. Think of it like a PowerPoint slide. Use these for announcements, titles, or any message you want to display without a background image.

Click "T+ Text Slide" in the Slides section

The slide editor opens in create mode.

Type your message and choose the style

Use the same controls as a text overlay: font, size, colour, position, and alignment.

Choose a slide background colour

Use the Slide background colour picker to set the background of the entire slide. Default is a dark navy.

Set the duration (optional)

How many seconds the slide displays before moving to the next one. Default is 10 seconds.

Click "Create slide"

The slide is added instantly — no processing needed. It appears in the list with a ready badge straight away.

Watermark & Logo

Each slideshow can display a small logo or watermark image in a corner of the screen on all slides — useful for branding or attribution. You can also control the watermark on individual slides, hiding it on some or showing it only on specific ones.

Uploading a watermark

Open the slideshow detail page and scroll to "Watermark / Logo"

This card sits between the Settings card and the Slides card.

Click "Upload Watermark"

Choose a PNG, JPG, or WebP image. Transparent PNG logos work best — they blend seamlessly over any slide.

Configure the appearance

Set position, size, and opacity using the controls that appear. The watermark is saved immediately — use "Save settings" to apply any configuration changes.

Controlling the watermark per slide

Once a watermark is uploaded, a small hexagon button (⬡) appears on each ready slide row. Click it to cycle through three states:

Button colourStateBehaviour
Grey ⬡ Inherited This slide follows the slideshow's default visibility setting.
Red ⬡ Hidden The watermark is hidden on this slide only, even if the slideshow default is "Show on all".
Blue ⬡ Forced on The watermark is shown on this slide only, even if the slideshow default is "Hidden by default".
ℹ
Common use cases:
• Watermark on all slides except one: set Default to "Show on all", then set that one slide to Red ⬡ (hidden).
• Watermark on one slide only: set Default to "Hidden by default", then set that one slide to Blue ⬡ (forced on).

Replacing or removing the watermark

To swap the logo, click Replace in the Watermark card and upload the new file. The old file is deleted automatically. To remove the watermark entirely, click Remove — this deletes the logo file and hides the watermark from all slides.

ℹ
Use a transparent PNG for the best results. A logo with a transparent background blends cleanly over both light and dark slides without an unsightly white box.
08

Publish & Disable

Every slideshow has a status — either Published or Disabled. Only published slideshows appear on the display. Disabled slideshows are completely hidden from viewers, regardless of any schedule settings, but you can still edit them freely in the admin panel.

Understanding the status buttons

The button shown tells you what will happen if you click it — and its colour clearly shows what action you're about to take:

ButtonCurrent statusWhat clicking does
Disabled Makes the slideshow live. It will appear on all displays immediately (subject to its schedule).
Published Removes the slideshow from all displays immediately. It can be re-published at any time.

Toggling from the detail view

You can also publish or disable a slideshow from inside its detail page. The status is shown in the Settings card alongside the same button. When a slideshow is disabled, a yellow warning banner is displayed at the top of the detail page as a reminder.

✓
Changes are instant. When you click Publish or Disable, all connected display screens update within 1–2 seconds via a live connection — no page refresh required on the display side.

Typical workflows

Setting up a new slideshow before going live: Create the slideshow (it starts disabled) → upload all your images, add text slides and overlays → adjust the order and schedule → click Publish when ready.

Seasonal or event content: Upload your Christmas/event content weeks in advance. Leave it disabled. On the day, click Publish. After the event, click Disable. The content remains on the server ready for next year — just click Publish again.

Pulling content quickly: If you need to immediately remove something from display (e.g. outdated information), click Disable on the relevant slideshow. The change takes effect within seconds on all screens.

09

Scheduling

Each slideshow has a schedule that controls when it appears. This only applies to published slideshows — disabled slideshows never appear regardless of schedule.

Schedule types

TypeBehaviour
Always active The slideshow shows whenever it is published, 24 hours a day, 7 days a week. This is the default for new slideshows.
Timed The slideshow only shows during a specific time window (e.g. 09:00–17:00) on selected days of the week. Outside this window it is hidden even if published.

Setting a timed schedule

Open the slideshow detail page and click Edit
Change Schedule from "Always active" to "Timed"

Fields for Start time, End time, and Active days will appear.

Set your start and end times

These are in 24-hour format. For example, 09:00 to 17:00 means the slideshow appears from 9 AM until 5 PM.

Select which days it should run

Tick the days of the week. Leave all ticked for every day.

Click Save, then Publish if it isn't already
⚠
The schedule uses the server Pi's clock. Make sure the Pi's time and timezone are set correctly. Run sudo raspi-config and check Localisation Options → Timezone if you notice timing is off.

What happens when no slideshows are active

If no published slideshow matches the current schedule (or there are no published slideshows at all), the display shows a plain black screen with a faint pulsing dot — this is intentional and indicates the display is connected and waiting. It is not an error.

10

Ordering & Priority

When more than one slideshow is published and scheduled at the same time, the system uses priority to decide which ones appear. Up to 5 slideshows can run simultaneously (their slides are merged into one continuous playlist in priority order).

Priority numbers

Lower numbers = higher priority. A slideshow with priority 1 appears before one with priority 2. When you create a slideshow, it gets the next available priority number automatically.

PriorityBehaviour
1Highest — slides from this slideshow appear first in the playlist.
2 – 4Slides are appended after higher-priority slideshows.
5+Included if up to the 5-slideshow limit hasn't been reached.
> 5 active at onceOnly the top 5 by priority number are shown. The rest are ignored until one of the top 5 becomes inactive.

Changing priority

Open the slideshow's detail page, click Edit in the Settings card, change the Priority number, and click Save.

Slide order within a slideshow

Inside a single slideshow, slides play in the order shown in the Slides card. Use the ↑ and ↓ buttons to reorder them. The playlist cycles continuously — after the last slide, it loops back to the first.

11

Remote Displays

Any device on the same network as the server can open the display page at http://<server-ip>:3000/ — including laptops, tablets, and additional Raspberry Pis. If you want to restrict which devices can see the display, use MAC filtering.

MAC filtering

MAC filtering lets you create an approved list of devices. When enabled, only approved devices can access the display and admin panel. Unapproved devices see a "Not Found" error.

ℹ
MAC filtering is disabled by default. You only need to enable it if you want to restrict which screens can show your content — for example, in a public-facing installation.

Enabling MAC filtering and adding a device

Find the MAC address of the device you want to approve

If the device already tried to connect and was blocked, a remote-display Pi will show its own MAC address on screen. On other devices, check the network settings or run ip link show in a terminal.

Go to Admin → Settings → MAC Filtering
Add the MAC address to the approved list

Enter the MAC address (e.g. aa:bb:cc:dd:ee:ff) and a label so you remember which device it is.

Enable MAC filtering

Toggle "MAC filtering enabled" to on. The server Pi itself (localhost) is always approved automatically.

⚠
Add your own device to the approved list before enabling MAC filtering. If you enable it without adding the browser you're using, you will be locked out of the admin panel. If this happens, SSH into the Pi, edit /opt/noticeboard/data/config.json, and set "enabled": false in the macFiltering section, then restart the service.
12

Settings

The Settings page in the admin panel contains system-wide options. Access it via the navigation bar.

SettingWhat it controlsDefault
Default slide duration How many seconds each image or text slide displays before advancing. This applies to all slides that don't have a custom duration set. Videos always play their full length. 10 seconds
Change password Update the admin panel password. Admin@12345
MAC filtering Enable/disable device approval, and manage the approved MAC address list. Disabled
Server port The port the server listens on. Changing this requires a server restart. Don't change this unless necessary. 3000

Restarting the server service

Some settings changes (e.g. port changes, or manual edits to config.json) require restarting the server. Run this command on the Pi via SSH or a terminal:

sudo systemctl restart noticeboard

Checking service logs

If something isn't working, the service logs are the best place to look for error messages:

sudo journalctl -u noticeboard -f

Press Ctrl+C to stop following the log.

13

Troubleshooting

Display shows a black screen

This is the most common issue and has several possible causes:

CauseHow to checkFix
No slideshows are published Open the admin panel. Do any slideshows show Published? Click Publish on a slideshow
Published slideshow has no ready slides Open the slideshow. Are any slides showing ready? Upload images, add text slides, or wait for processing to complete
Schedule doesn't match current time Check the slideshow's schedule setting Change schedule to "Always active" or adjust the time window
Server not running Can you open the admin panel? Is the URL reachable? Run sudo systemctl restart noticeboard
NODE_ENV not set to production Run cat /opt/noticeboard/.env — does it contain NODE_ENV=production? Run: echo "NODE_ENV=production" >> /opt/noticeboard/.env then restart the service

Text overlay or text slide doesn't appear on screen

Check that the slideshow is published and the schedule is active. For text overlays, open the slide editor (✏) and confirm the text field is not empty and the position sliders are not pushed completely off the edge (x and y both at 0 or 100 can push the text out of view). For text slides, confirm the slide shows ready in the Slides card.

Watermark logo isn't showing on the display

Check the following in the Watermark / Logo card: (1) A logo file has been uploaded — a preview image should appear. (2) Default visibility is set to "Show on all slides" (or the specific slide's ⬡ button is blue/forced on). (3) Click "Save settings" after making any changes. If it still doesn't appear, re-publish the slideshow to force a playlist refresh.

Admin panel shows "Session expired" or logs me out

Simply log in again. Sessions expire after a period of inactivity. If you are repeatedly logged out very quickly, restart the server — the JWT secret is regenerated on a fresh install and that can occasionally cause stale session cookies.

sudo systemctl restart noticeboard

Uploaded image/video doesn't appear after processing

Check that the slideshow is published and that the schedule is currently active. If the slide shows error, delete it and try uploading again. For videos, check that ffmpeg is installed on the Pi:

ffmpeg -version

If it's not found, install it:

sudo apt-get install -y ffmpeg

Remote display Pi won't connect / shows MAC address screen

The display Pi is connected and working, but MAC filtering is blocking it. You need to add its MAC address to the approved list in Admin → Settings → MAC Filtering. The MAC address is shown on the display Pi's screen.

Display is connected but slides seem slow to update after a change

The scheduler checks for schedule changes every 60 seconds. If you change a schedule setting, wait up to 1 minute for the display to update. Changes to slide content (uploading, reordering, adding text slides, publishing/disabling) take effect within a few seconds.

The Pi display is blank or black on boot (kiosk doesn't start)

The kiosk waits for the server to be ready before launching Chromium. If the server takes longer than usual to start (e.g. after a system update), the kiosk should still launch — just more slowly. Check the server is running:

sudo systemctl status noticeboard

If the service is failed, check the logs for why:

sudo journalctl -u noticeboard --no-pager -n 50

How to check everything is working

From the Pi's terminal, this one-liner confirms the server is up and accepting connections:

curl -s -o /dev/null -w "HTTP %{http_code}\n" http://localhost:3000/
# Should print: HTTP 200
💡
Still stuck? Check the project on GitHub at github.com/fructus-sum/noticeboard for the latest updates, or file an issue with the output from sudo journalctl -u noticeboard -n 100 attached.
